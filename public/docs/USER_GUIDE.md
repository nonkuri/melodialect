# Melodialect ユーザーガイド

対象バージョン: v1.0.1

[READMEへ戻る](https://github.com/nonkuri/melodialect#readme) | [パラメーターリファレンス](https://github.com/nonkuri/melodialect/blob/main/docs/PARAMETERS.md)

このガイドは、ブラウザ版 Melodialect で曲を生成し、候補を比較し、必要な部分を編集して書き出すまでの操作を説明します。各設定値が曲へ与える影響は、[パラメーターリファレンス](https://github.com/nonkuri/melodialect/blob/main/docs/PARAMETERS.md)を参照してください。

## 1. 起動する

### 公開版

[公開版 Melodialect](https://nonkuri.github.io/melodialect/) を開きます。一般利用に Node.js、npm、JSON編集環境、サーバーは不要です。

初回案内では **サンプル曲を開く** を選べます。通常画面の左上にある **目的から始める** からも、「まず1曲を完成」「静かなバラード」「グルーヴを作る」「4〜5分のフル構成」を選べます。プリセットを選んだ後に **♪ 全体生成** を押してください。

### ソースから起動する開発者

開発またはCLI利用では Node.js 22 以上が必要です。初回だけリポジトリのフォルダーで次を実行します。

```powershell
npm install
```

起動するときは次を実行します。

```powershell
npm run dev
```

ターミナルに表示されたURL、通常は [http://localhost:5173](http://localhost:5173) をブラウザで開きます。終了するときはターミナルで `Ctrl+C` を押します。

### インストール、オフライン利用、更新

- **Chrome**: 右上メニューの「キャスト、保存、共有」→「ページをアプリとしてインストール」、またはアドレスバーのインストールアイコンを使います。
- **Edge**: アドレスバーの「アプリを利用できます」アイコンからインストールします。
- **Firefox for Windows**: アドレスバーのWebアプリボタンからタスクバーへ追加します。macOS / Linuxでは通常のタブまたはブックマークを使います。
- **Safari on macOS**: 共有ボタンまたは「ファイル」→「Dockに追加」を使います。
- **iPhone / iPad**: Safariの共有メニューから「ホーム画面に追加」を使います。

初回オンライン読込時にアプリ本体、Worker、同梱標準音源を保存します。準備後は通信を切っても同じURLから起動できます。ユーザー音源も取り込み済みなら端末内から読まれます。

新しい版が見つかると画面上部に通知します。編集中の内容が「保存済み」になったことを確認して **保存済み・更新する** を押してください。更新準備に失敗した場合の **アプリキャッシュだけ修復** はアプリ本体だけを取り直し、プロジェクトとユーザー音源を削除しません。

### アプリ、プロジェクト、音源の保存先

| データ | 内容 | 更新/キャッシュ修復 | ブラウザのサイトデータ削除 |
|---|---|---|---|
| アプリキャッシュ | 画面、コード、Worker、標準音源 | 新版へ入替 | 削除 |
| プロジェクト | 曲、設定、編集、保存世代 | 保持 | 削除 |
| ユーザー音源 | 取り込んだSF2本体と索引 | 保持 | 削除 |

サイトデータを消す前に、**プロジェクト一覧 → 全曲バックアップ** と、必要なダイアレクトJSONを書き出してください。SF2原本も別の場所に保管してください。サイトデータ削除、公開URLの変更、別ブラウザへの移動後はSF2の再取り込みが必要です。

### ブラウザごとの違い

| ブラウザ | インストール | SF2選択/書き出し | 保存データ |
|---|---|---|---|
| Chrome | Windows / macOS / Linuxで可 | 標準ファイル選択とダウンロード | Chromeのプロファイルごと |
| Edge | Windows / macOSで可 | 標準ファイル選択とダウンロード | Edgeのプロファイルごと |
| Firefox | WindowsでWebアプリ可。macOS / Linuxはタブ利用 | 標準ファイル選択とダウンロード | Firefoxのプロファイルごと |
| Safari | macOS 14以降はDock追加、iOS/iPadOSはホーム画面追加 | 標準ファイル選択とダウンロード | SafariとmacOS Webアプリは別領域 |

保存可能量はブラウザの判断と端末の空き容量で変わります。プライベート/シークレットモードでは永続保存を前提にしないでください。Melodialectは File System Access API だけに依存せず、通常のファイル選択を使うため、各対応ブラウザでJSONとSF2を読み込めます。

## 2. 画面の構成

| 場所 | 主な役割 |
|---|---|
| 最上段 | ピアノロール／譜面の切り替え、歌詞編集、Undo／Redo、全体生成、MIDI／WAV／WAVステム／MusicXML／テキスト書き出し |
| プロジェクトバー | 曲名、新規作成、保存、最近使った曲、JSON入出力、バリエーション比較 |
| トランスポート | 再生、一時停止、停止、再生位置、範囲ループ、メトロノーム、カウントイン |
| 構成バー | 選択中セクションの種類、ダイアレクト、小節数、移調、BPM、追加／複製／削除 |
| 左ペイン | 曲全体のダイアレクトとその管理、キー、調性、拍子、テンポ、構成、ループ、シード |
| 中央上部 | 選択セクションのロック、部分再生成、コード進行設計、ノート／コード編集 |
| 中央 | 適用前に調整できる伴奏・作曲パラメーター、曲全体の表情設計、即時反映されるミキサー、ピアノロールまたは譜面、生成根拠 |
| 最下段 | セクション選択、頭出し、範囲ループ指定、ドラッグによる並べ替え |

「伴奏アレンジ／ミキサーと音色／作曲パラメーター」とピアノロールの間、およびピアノロールと「生成根拠の解説」の間にある横線は、上下へドラッグして各表示領域の高さを調整できます。仕切りをダブルクリックすると初期値へ戻ります。

## 3. 最初の1曲を作る

1. 左ペインの **ダイアレクト** を選びます。選択時に、そのダイアレクトの推奨キー、調性、テンポ、拍子、伴奏設定が読み込まれます。
2. 必要なら **キー**、**調性**、**拍子**、**テンポ** を変更します。
3. **構成** を選びます。最初は `Verse-Chorus ×2` のままで構いません。
4. 結果を後から再現したい場合は **シード** を指定します。🎲はランダムなシードを設定します。
5. 右上の **♪ 全体生成** を押します。
6. **▶ 再生** で試聴します。
7. 気に入ったら MIDI、WAV／WAVステム、MusicXML、テキストのいずれかで書き出します。

ダイアレクトなど左ペインの設定は、変更しただけでは現在の曲へ反映されません。**♪ 全体生成** を押した時点で新しい曲へ反映されます。

## 4. おすすめの作業順

手動編集をできるだけ失わずに作業するには、次の順序がおすすめです。

1. ダイアレクト、キー、テンポ、構成、シードを決めて全体生成する
2. 伴奏アレンジと作曲パラメーターを調整する
3. バリエーションを作り、候補を比較する
4. 部分再生成でセクションを詰める
5. ノートとコードを手動編集し、残したい箇所をロックする
6. ミキサーと音色で最終バランスを整える
7. JSONでバックアップし、MIDI／WAV／テキストを書き出す

> **重要:** 伴奏アレンジ、作曲パラメーター、構成編集は曲を再構築します。細かなノート編集より先に決めてください。ミキサーの音量、パン、音色は生成済みノートを変更しないため、最後に調整できます。

### 編集パネルとミキサー

伴奏アレンジと作曲パラメーターは、値を動かしただけでは現在の曲を変更しません。内容を確認して **適用** を押すと再構築され、**キャンセル** で適用済みの値へ戻せます。未適用の値がある状態で **♪ 全体生成** を押した場合は、画面に表示中の値を使って曲全体を生成し、その値を適用済みにします。ミキサーのM、S、Vol、Pan、音色、Masterは再生音へ即時反映されます。

**新規保存** では、Master音量とピーク保護に加え、全パートのミュート、ソロ、音量、パン、内蔵音色、SoundFont割り当てをプリセットとして保存します。プリセットを選ぶとこれらをまとめて復元し、選択中のプリセットは **上書き** または **削除** できます。同じ名前で新規保存した場合も、確認後に既存プリセットを上書きします。

## 5. 生成とバリエーション

### 全体生成

**♪ 全体生成** は、左ペインの設定と画面に表示中の伴奏・作曲パラメーターを使って曲全体を作り直します。選択したダイアレクトと曲の土台のテンポはこの時点で反映されます。セクション別BPMを明示的に変更した箇所はその値を保ち、元の基準テンポを継承していたセクションだけ新しいテンポへ更新します。全体生成時にはロックが解除され、セクション個別の再生成履歴もリセットされます。生成前の状態はバリエーション履歴へ追加され、成功するとボタンに `✓ Modal 生成済み` のような完了表示が一時的に出ます。

現在の曲と同じシードのまま全体生成を繰り返した場合は、無反応に見えないよう新しいランダムシードへ自動更新して別候補を生成します。手入力または🎲で現在と異なるシードを選んだ場合は、その値を使用します。生成前スナップショットをブラウザへ保存できない場合も、曲の生成自体は続行します。保存状態の警告が出たときはJSONバックアップを作成してください。

### バリエーション

**＋ バリエーション** はランダムな新しいシードで候補を作り、現在の候補と新候補を履歴へ保存します。履歴のプルダウンから以前の候補を読み込めます。

- 最大24候補が新しい順に保持されます。
- ☆／★で最新候補のお気に入り状態を切り替えられます。
- バリエーションには生成設定、ミキサー、編集内容、ロックも含まれます。
- シード履歴は画面右下に表示されます。

### 部分再生成

最下段のタイムラインで対象セクションを選び、中央上部の再生成ボタンを使います。

| ボタン | 変わる内容 |
|---|---|
| 全体 | 選択セクションのメロディ、コード、伴奏 |
| メロディ | 選択セクションのメロディだけ |
| コード | 選択セクションのコードだけ |
| 伴奏 | ピアノ、ベース、ギター、ドラム |

他のセクションは変更されません。ロックされたセクション、小節、ノート、コードは部分再生成から保護されます。

### 目的別の作曲ツール

追加条件は一つの画面にまとめず、作業する場所に近い四つの入口から設定します。

- **コード進行…（中央上部）** — `I:4 | vi:4 | _:4 | V7:4` のようにローマ数字と拍数を入力します。標準生成、入力固定、空欄だけ補完、リハーモナイズ候補を切り替えられます。拍数不足・重複・不正記号はエラー、調性・終止・ダイアレクト語彙との違いは警告として表示されます。
- **部分生成** — 固定コードを変えず、選択ダイアレクトの規則でメロディだけ、または伴奏だけを作り直せます。
- **比較と再利用** — リハーモナイズ前の原案は保持され、候補から復元できます。コード進行はキー相対のローマ数字テンプレートとして保存され、別キーへ移調して再利用できます。
- **曲全体の表情…（編曲・作曲調整）** — ピアノロールで選択したメロディ断片を固定し、Chorus間の変奏量、セクション別の緊張度・密度・明るさ・終止形を設定します。現在の音域・密度・明るさは相対バーで比較できます。
- **歌詞を編集（最上段）** — 日本語、英語、スキャットを選び、行単位の歌詞と音符ごとの音節を編集できます。セクションのコピー／再生成と、MusicXML／テキストへの反映に対応します。
- **管理（左ペインのダイアレクト欄）** — 内蔵/保存済み定義の複製、フォーム編集、全項目検証、試し生成、保存・再利用、JSON入出力を行います。

コード設定・表情設定・歌詞は、それぞれの画面の保存ボタンでプロジェクトへ保存されます。コード制約を使わない場合は生成モードを「標準」のままにします。

### ダイアレクトをフォームで作る

1. 左ペインのダイアレクト欄で **管理** を開きます。
2. 内蔵または保存済みダイアレクトを選び、**複製して新規作成** を押します。
3. ID、名前、推奨キー/調性/テンポ/拍子、コード語彙と優先進行、旋律輪郭/跳躍/反復、フレーズ長、推奨伴奏を編集します。
4. 項目下のエラーを修正し、**試し生成** で固定シード100のVerse/Chorusとコード進行を確認します。
5. **端末へ保存** の後、**この曲で使う** を押し、画面へ戻って **♪ 全体生成** します。

保存済み一覧では複製、編集、名前変更、削除、JSON書き出しができます。JSON読込も同じ全項目スキーマで検証します。基本GUIに表示しないリズムテンプレート、カデンツ、セクション規則、転調などは、複製・編集・保存しても失われません。

## 6. ロックの使い分け

| ロック | 対象 | 使いどころ |
|---|---|---|
| セクション | 選択中のセクション全体 | 採用済みのVerseやChorusを完全に残す |
| 小節 | 選択ノートまたはコードが属する小節とパート | 手直しした1小節を残す |
| ノート | 選択中の1音 | 特定の決め音だけ残す |
| コード | 選択中の1コード | 和声の要所だけ残す |

ノートの移動・伸縮・追加、ツールバーの1/16整列・削除、コードの置換・挿入・削除を行うと、その小節が自動的にロックされます。

ロックは部分再生成に対して有効です。**全体生成** や曲全体を再構築するパラメーター変更の保護には使えません。

## 7. 構成を編集する

最下段のタイムラインでセクションを選ぶと、上部の構成バーで次を編集できます。

- **種類** — Intro、Verse、Chorus、Bridge、Outro
- **ダイアレクト** — そのセクションだけに使う作曲スタイル
- **小節** — 1〜32小節
- **移調** — -12〜+12半音
- **BPM** — 40〜240。セクションごとのテンポ変更
- **＋追加** — 選択セクションを基準に新しいVerseを末尾へ追加
- **複製** — 選択セクションを直後へ複製
- **削除** — 選択セクションを削除。最低1セクションは残ります

タイムラインのブロックをドラッグ＆ドロップすると、セクションの順番を変更できます。構成を変更すると曲全体が再構築され、既存ロックは解除されます。

### カスタム構成

左ペインの **カスタム構成を追加** では、次の略号をカンマ区切りで入力します。

| 略号 | セクション |
|---|---|
| `i` | Intro |
| `v` | Verse |
| `c` | Chorus |
| `b` | Bridge |
| `o` | Outro |

例: `i,v,c,v,c,b,c,o`

最大16セクションまで登録でき、カスタム構成はブラウザに保存されます。`v:modal,c:twilight` のようなダイアレクト指定も内部形式として利用できますが、通常は合作モードのプルダウンを使う方が簡単です。

### 合作モード

**セクション別ダイアレクト** で、各セクションへ別のダイアレクトを割り当てられます。`メインと同じ` を選ぶと、上部で選んだ曲全体のダイアレクトを使用します。

## 8. ピアノロールを編集する

ピアノロールには全パートが色分け表示されます。

| 色 | パート |
|---|---|
| オレンジ | メロディ |
| 青 | ピアノ |
| 黄 | ギター |
| 緑 | ベース |
| 紫 | ドラム |

### 基本操作

- 空白をクリック: その位置へ再生位置を移動
- 空白をダブルクリック: 1拍のメロディノートを追加
- ノートをクリック: ノートを選択
- ノートをドラッグ: 位置と音高を移動
- ノート右端をドラッグ: 長さを変更
- コード名をクリック: コードを選択

ツールバーでは横・縦ズーム、1/4・1/8・1/16・三連符グリッド、表示パートを切り替えられます。ドラッグ範囲選択と `Ctrl`／`Cmd` を使った複数選択に対応し、コピー、切り取り、貼り付け、複製、まとめて移調、クオンタイズ、削除を実行できます。下部のベロシティレーンでは選択ノートの強さを編集できます。

**再生位置を追従** をオンにすると、演奏中の再生位置が表示範囲から外れる前にピアノロールが自動スクロールします。オフにすると編集中の表示位置を維持します。横スクロールバーは曲の完全な先頭まで戻せます。

ノートは16分音符、つまり0.25拍単位で移動・伸縮します。選択時にはツールバーから -12、-1、+1、+12半音の移動、**1/16整列**、削除、小節／ノートロックを実行できます。

コード選択時はローマ数字で入力し、**置換**、**挿入**、**削除**を実行できます。入力例は `I`、`vi`、`V7`、`IV△7`、`♭VII` です。コードを変更すると、そのセクションの伴奏も更新されます。

## 9. 譜面と生成根拠を見る

### 譜面表示

上部の **譜面** を押すと、メロディとコードネームをリードシート形式で表示します。**仮歌詞を表示** を有効にすると、生成された仮歌詞も表示されます。ピアノロールへ戻るには **ピアノロール** を押します。

### 生成根拠

**生成根拠の解説** を開くと、各セクション・小節で適用されたルールを確認できます。ダイアレクト固有の進行、モチーフ、グルーヴ、部分再生成などを確認すると、意図した結果へ調整しやすくなります。

編集パネルと譜面／ピアノロールの間、および譜面／ピアノロールと生成根拠の間の仕切りは、上下へドラッグできます。ダブルクリックで初期値へ戻ります。

## 10. 再生とループ

- **▶ 再生** — 現在位置から再生
- **Ⅱ 一時停止** — 現在位置を保持して停止
- **■ 停止** — 停止して曲頭へ戻る
- **再生位置** — 曲頭を含む任意位置へ0.25拍単位で頭出し
- **範囲ループ** — 開始小節から終了小節までを繰り返す
- **メトロノーム** — 再生中に拍クリックを鳴らす。小節頭は高い音
- **1小節カウント** — 再生前に1小節分のクリックを鳴らす

最下段のセクションをクリックすると、そのセクションの先頭へ移動します。ダブルクリックすると、そのセクションを範囲ループに設定します。

左ペインの **ループモード** は再生範囲のループとは別機能です。曲の終止を半終止に変え、曲末から曲頭へ自然につながる曲を生成します。

## 11. 保存、バックアップ、書き出し

### ブラウザ内の保存

曲は編集後約0.45秒でブラウザのローカルストレージへ自動保存されます。**保存** ボタンで即時保存もできます。最近使った曲は最大8件表示されます。

> ブラウザのサイトデータを削除すると、ブラウザ内のプロジェクトとカスタム構成も消えます。重要な曲はJSONでバックアップしてください。

### JSON

**JSON** は曲名、設定、生成結果、手動編集、ロック、バリエーションを含むプロジェクトファイルを保存します。**読込** で読み込むと、元プロジェクトを上書きせず、曲名に `(読込)` が付いた別プロジェクトとして開きます。

### MIDI

メロディ、コードに基づく伴奏、テンポ、拍子、セクション情報を含むMIDIファイルを出力します。DAWで音源を差し替えて編集する用途に向きます。

### WAV

ブラウザ再生と同じ内蔵シンセ、ミキサー、音色でWAVをレンダリングします。曲が長い場合は書き出しに少し時間がかかります。

### WAVステム

メロディ、ピアノ、ギター、ベース、ドラムを、同じ開始位置・長さ・サンプルレートの個別WAVとして書き出します。DAWでパートごとに処理するときに使用します。

### MusicXML

メロディ、拍子、テンポ、コードネームをMusicXMLとして書き出します。対応する楽譜ソフトで開き、譜面の清書や編曲へ引き継げます。

### テキスト

外部の音楽生成サービスなどへ貼り付けるスタイルプロンプト、仮歌詞、コード進行をまとめたテキストを出力します。仮歌詞はプレースホルダーなので、必要に応じて差し替えてください。

## 12. キーボードショートカット

| キー | 動作 |
|---|---|
| `Space` | 再生／一時停止 |
| `Ctrl+Z` / `Cmd+Z` | Undo |
| `Ctrl+Shift+Z` / `Cmd+Shift+Z` | Redo |
| `Ctrl+Y` / `Cmd+Y` | Redo |
| `Delete` / `Backspace` | 選択ノートを削除 |

入力欄やプルダウンへフォーカスがある間は、誤操作を避けるためショートカットは動作しません。Undo／Redoの履歴はページを再読み込みすると消えます。

## 13. よくある問題

### 設定を変えても曲が変わらない

左ペインのダイアレクト、キー、調性、拍子、テンポ、構成、シードは **♪ 全体生成** を押したときに反映されます。成功時はボタンに生成済みのダイアレクトが表示されます。伴奏アレンジと作曲パラメーターは値を調整した後、**適用** または **♪ 全体生成** を押すと再構築され、ミキサーは再生音へ即時反映されます。

### 手動編集した音が消えた

全体生成、構成編集、伴奏アレンジ、作曲パラメーターの変更は曲を再構築します。手動編集はこれらを決めた後に行い、候補はバリエーションまたはJSONへ保存してください。

### 音が出ない、特定パートだけ聞こえない

ブラウザやOSの音量に加え、ミキサーの `M`、`S`、`Vol` を確認してください。どれか1パートがソロになっていると、他のパートは聞こえません。

### ループ範囲がおかしい

開始と終了が曲の小節数内にあり、開始が終了以下になっているか確認してください。セクションをダブルクリックすると正しい範囲を簡単に設定できます。

### 前のプロジェクトが見つからない

プロジェクトはブラウザごとのローカルストレージへ保存されます。別ブラウザ、シークレットウィンドウ、別PCでは共有されません。JSONファイルを使って移動してください。

### 保存または読込に失敗した

画面最下部の赤い保存状態、または管理画面のエラーを確認します。容量不足なら不要なプロジェクト/音源を整理する前に全曲バックアップを保存してください。読込失敗ではファイル形式と対象バージョンを確認し、元ファイルを上書きせず残してください。保存済みプロジェクトがない画面にはサンプル曲を開く導線が表示されます。

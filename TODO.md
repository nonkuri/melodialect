# Melodialect TODO

作曲アプリとして、生成後の「聴く → 試す → 直す → 比較する → 保存する」を、安全かつ効率よく完結させるためのロードマップ。

## 優先度の定義

- **P0** — データ消失や編集の巻き戻りを防ぐため、他機能より先に必要
- **P1** — 日常的な作曲・編集効率を大きく改善する中核機能
- **P2** — 音質、外部連携、出力品質を改善する機能
- **P3** — 作曲表現と拡張性を広げる機能
- **Later** — 価値はあるが、コア体験の完成後に判断する機能

## 現時点の製品方針

- v0.6でデータ保護と非破壊編集を固めた後、**編集機能の拡充より先に音質と外部連携を改善する**
- v0.7を「音質と外部連携」、v0.8を「編集操作の完成」とする
- 最初の一般向け配布形態は**PWA**とし、まずGitHub Pagesへ公開する。独自サイトや常時稼働サーバーは前提にしない
- 公開版の利用者はURLをブラウザで開くかPWAをインストールするだけとし、**Node.js、npm、開発環境の導入を要求しない**
- Node.jsを必要とするのは、ソースから起動する開発者とCLI利用者だけとする
- 楽曲、設定、ユーザー音源は原則として端末内で処理し、ユーザーが明示的に共有しない限りサーバーへ送信しない
- Electron版はPWAと並行開発せず、大容量音源、SFZ、ファイルシステム、外部MIDIなどPWAの制約が実利用上の問題になった場合に追加する

## スコープ外

Melodialectは生成結果を整えてDAWや楽譜ソフトへ渡す作曲支援ツールとし、次は当面実装しない。

- マルチトラック録音
- オーディオ波形編集
- VST／Audio Unitプラグインのホスト
- 本格的なミキシングエフェクトとマスタリング
- DAWと同等のオートメーション編集

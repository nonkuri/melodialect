# Melodialect 仕様書

**作曲家の「音楽的方言」をルールベースで再現する楽曲ジェネレーター**

- バージョン: 0.1(ドラフト)
- 作成日: 2026-07-18
- 参考: [Claude Fable 5 によるビートルズ風楽曲ジェネレーターの記事(TechnoEdge)](https://www.techno-edge.net/article/2026/07/13/5285.html)

---

## 1. コンセプト

機械学習モデルを使わず、**音楽理論に基づく明示的なルール(=作曲家ごとの「方言」)** をコードとして実装し、それらを組み合わせて楽曲(コード進行+メロディ+伴奏)を生成するブラウザアプリ。

- 学習データ不要・著作物の複製なし。生成されるのは「技法・傾向」を模した完全オリジナル曲
- ルールがすべてコードで可視化されているため、「なぜこの音になったか」を説明できる(生成根拠の解説機能)
- 生成結果は WAV / MIDI(SMF) で書き出し、Logic Pro 等の DAW や Suno 等の外部サービスに持ち込める

## 2. 用語

| 用語 | 意味 |
|---|---|
| ダイアレクト(Dialect) | 作曲家 1 人分の作曲スタイル定義。コード進行・メロディ輪郭・リズム傾向のルールセット |
| セクション | Verse / Chorus / Bridge / Intro / Outro などの楽曲構成単位 |
| ソング | セクションの並び+キー+テンポ+ダイアレクト割り当ての全体 |

## 3. 対象ユーザーとユースケース

1. **DTM ユーザー**: 作曲の叩き台(コード進行+メロディ)を生成し、MIDI で DAW に取り込んで仕上げる
2. **音楽理論の学習者**: 「モーダル・インターチェンジを使うとどう響くか」を、ルールの ON/OFF で聴き比べて学ぶ
3. **AI 歌声サービス利用者**: 生成した曲+仮歌詞を Suno 等に渡してボーカル入りデモを作る

## 4. 機能要件

### 4.1 ダイアレクト(スタイルプリセット)

9 種類 (D1〜D9) を実装する。各ダイアレクトは以下の 6 層で定義する:

1. **コード進行** — 語彙+マルコフ遷移に加え、3〜4 コードの定型句(イディオム)、終止形の好み(全終止/変格終止/偽終止/モーダル終止)、ハーモニックリズム(2 小節 1 コード〜1 小節 2 コード)
2. **リズム語彙** — ダイアレクト別の重み付きリズムテンプレート(シンコペーション・休符を含む)とアウフタクト(先取音)の傾向
3. **モチーフ** — 冒頭フレーズのリズムと輪郭を記憶し、以降のフレーズで反復・移調反復(シークエンス)する確率
4. **メロディ輪郭** — 跳躍確率・跳躍幅・跳躍後バイアス・同音連打・逆ペダル
5. **非和声音とセクション対比** — 倚音・掛留・半音階経過音の確率、セクション別の音域シフトとサビの音数バイアス
6. **転調傾向** — ブリッジ等での転調確率と移調量

#### D1: Modal(モーダル進行+語りの同音連打)
- コード: モーダル・インターチェンジ多用(♭VII, ♭VI, iv)。♭VI→♭VII→I の定型句、♭VII→I のモーダル終止。2 小節 1 コードのゆったりしたハーモニックリズム
- メロディ: 反復音・狭い音域・同音連打。休符を挟む語りのリズム。モチーフの執拗な反復 (0.75)
- 構成: フレーズ長の変則化(7 小節など)。サビでも音域がほぼ上がらないフラットなダイナミクス

#### D2: Chromatic(半音階クリシェ+歌心のモチーフ反復)
- コード: 半音階クリシェ(C → C/B → C/B♭ → F/A)。I→vi→ii7→V7 の定型句、全終止中心
- メロディ: 順次進行主体で歌いやすく、跳ねる付点リズムとアウフタクト。モチーフ反復 (0.6) と倚音
- ベース: メロディックなウォーキング/対旋律的な動き。サビは音域 +3 半音・音数増

#### D3: Pedal(逆ペダル+掛留のドローン)
- コード: 逆ペダルポイント(メロディ固定・コードのみ変化)。I→♭VII→IV→I、変格終止 (IV→I) 中心
- メロディ: ペダル音を軸にした滞留と掛留 (サスペンション 0.2)。長い音価のドローンリズム
- その他: II7(ダブルドミナント)の経過的使用

#### D4: Twilight(哀愁進行+大跳躍シンコペーション)
- メロディ: **フレーズ頭で大きく上方跳躍し、その後なだらかに下降**する輪郭
  - 跳躍確率: 通常フレーズ 28%、サビ頭 95%
  - 跳躍幅: 7〜10 半音(完全5度〜短7度)
  - 跳躍直後は下降方向にバイアス。倚音 (0.22) の多い哀愁の節回し
- コード: メジャー7th を含むソフトな響き。IV△7→III7→vi の哀愁定型句 (0.45)、サビは 1 小節 2 コードに細分
- リズム: シンコペーションと裏拍入り、アウフタクト多用 (0.5)。サビは音域 +4 半音・音数増
- 転調: ブリッジで下属調/属調へ転調しやすい (0.6)
- テンポ/性格: スローバラード寄りのデフォルト値

#### D5: Angular(七の和音+変則フレーズの跳躍ポップ)
- コード: △7/m7 主体の洗練和声、iii7→vi→ii7→V7 の定型句、ii7→I△7 のソフトな終止。1 小節 2 コード多め
- メロディ: 跳躍が多い角張った輪郭 (0.22/サビ頭 0.6) と倚音 (0.18)。シンコペのリズム語彙とアウフタクト (0.4)
- 構成: 変則フレーズ確率 0.35 (7 小節・6 小節)。ブリッジで全音上へ転調しやすい

#### D6: Orchestral(転回ベース+浮遊和声のポップ)
- コード: 半音階クリシェ (転回ベース) + ♭VI→♭VII→I のリフト、変格終止と偽終止傾向 (V7→vi 0.4)
- メロディ: 順次進行主体 (メロディックベース) に掛留 (0.15)。ゆったりした音価のリズム
- 転調: ブリッジで全音上/下属調へ (0.55)。サビ +3 半音のレジスタ

#### D7: Bossa(シンコペ和声+ささやきの同音連打)
- コード: ii7→V7→I△7 連鎖 (0.45) と II7 (ドッペルドミナント) 経由。1 小節 2 コードのシンコペ和声 (0.4)
- メロディ: 狭い音域の同音連打 (0.35) と半音階経過音 (0.15)。付点と裏拍のボサリズム
- 性格: サビでも控えめな対比 (密度バイアス 0.1)。ブリッジで全音下/下属調へ (0.35)

#### D8: Ostinato(機械的リフレイン+ミニマル短調)
- コード: 短調 (デフォルト A minor)。i / VI / VII 中心のミニマル語彙、2 小節 1 コード (0.4)、VII→i のモーダル終止
- メロディ: 均質な 8 分連打 (同音連打 0.5)、モチーフ反復 0.85 のシーケンサ的ループ。跳躍・非和声音・アウフタクトなし
- 構成: 変則なしの機械的な規則性。セクション対比もフラット

#### D9: Serene(揺れる△7+静謐な 3 拍子)
- 拍子: **デフォルト 3/4** (ダイアレクト別デフォルト拍子)。テンポ 66 の静かなワルツ
- コード: I△7↔IV△7 の揺れ (定型句 0.45)、2 小節 1 コード (0.35)、ii7→I△7 / IV△7→I△7 の柔らかい終止のみ (V7 をほぼ使わない)
- メロディ: 1 拍目を休符にして 2 拍目から入る疎らなリズム語彙、掛留 (0.18)、モチーフ反復 0.7。サビとの対比はほぼなし (+1 半音)

> ダイアレクトは JSON ベースの宣言的定義とし、後から追加・ユーザー編集できる設計にする(§6.2)。

### 4.2 楽曲生成

| 項目 | 内容 |
|---|---|
| キー | 12 キー × メジャー/マイナー。デフォルトはダイアレクトごとの推奨キー |
| テンポ | 40〜200 BPM。ダイアレクトごとのデフォルト値あり |
| 拍子 | 4/4 / 3/4 / 6/8(6/8 は複合拍子として 1 小節 = 付点4分 × 2 拍で扱う) |
| セクション構成 | Intro / Verse / Chorus / Bridge / Outro を並べて指定。プリセット(例: V-C-V-C-B-C)あり |
| セクション別ダイアレクト割り当て | 「Verse は Modal、Chorus は Chromatic」のような合作モードをサポート |
| シード | 乱数シード指定で同じ曲を再現可能 |

生成パイプライン(すべて決定的ルール+シード付き乱数):

1. **構成生成**: セクション列とフレーズ長(変則小節数を含む)を決定
2. **コード進行生成**: ダイアレクトのコードルール(借用和音・クリシェ・ペダル等)をマルコフ的な遷移表+制約で適用
3. **メロディ生成**: コードトーン/テンションを土台に、輪郭ルール(跳躍確率・跳躍幅・下降バイアス)でピッチ列を生成し、リズムテンプレートに割り付け
4. **伴奏生成**: 選択された伴奏パターンにコードを展開
5. **仮歌詞生成**: 音節数に合わせたラ・ラ・ラ/日本語ダミー歌詞を割り当て(任意)

### 4.3 伴奏パターン

- 楽器: ピアノ / アコースティックギター(アルペジオ・ストローク) / エレキベース / 簡易ドラム
- 各楽器 4 パターン以上(例: ピアノ = ブロックコード、アルペジオ、8 分刻み、バラードパターン)
- ダイアレクトごとに推奨の組み合わせをプリセット化

### 4.4 再生・表示

- **再生**: Web Audio API によるブラウザ内再生(再生/停止/ループ/セクション頭出し)
- **譜面表示**: メロディ+コードネームのリードシート表示(VexFlow 等)。再生位置のハイライト
- **ピアノロール表示**: メロディと伴奏をトラック別に表示
- **生成根拠の解説表示**: 「3 小節目: ♭VII はモーダル・インターチェンジ(D1 ルール 2)」のように、各小節に適用されたルールを注記できる

### 4.5 エクスポート

| 形式 | 内容 |
|---|---|
| MIDI (SMF Format 1) | トラック別(メロディ/ピアノ/ギター/ベース/ドラム)、テンポ・拍子・コードネーム(マーカー)入り。DAW 取り込み用 |
| WAV | ブラウザ内レンダリング(OfflineAudioContext)による 44.1kHz/16bit ステレオ |
| テキスト | コード進行+仮歌詞のテキスト出力(Suno 等へのコピペ用スタイルプロンプト付き) |
| 解説動画(将来) | コード進行とメロディラインを可視化した動画(WebCodecs / キャンバス録画) |

### 4.6 プロジェクト保存

- 生成設定+シードを JSON としてローカル保存/読み込み(localStorage + ファイルダウンロード)
- サーバー不要。完全クライアントサイドで動作

## 6. 技術仕様

### 6.1 スタック

| 層 | 技術 |
|---|---|
| 言語 | TypeScript |
| UI | React + Vite |
| 再生/合成 | Web Audio API(自前シンセ or サウンドフォント再生。Tone.js 採用可) |

- 外部 AI API は使わない(仮歌詞の高度化で将来オプション化する余地のみ残す)

### 6.2 ダイアレクト定義フォーマット(骨子)

```jsonc
{
  "id": "twilight-ballad",
  "name": "Twilight (哀愁進行+大跳躍シンコペーション)",
  "defaults": { "key": "F", "mode": "major", "bpm": 72 },
  "chord": {
    "vocabulary": ["I△7", "IV△7", "III7", "vi", "ii7", "V7", "♭VII"],
    "transitions": { "IV△7": { "III7": 0.5, "I△7": 0.3, "ii7": 0.2 } },
    "idioms": [                                // 3〜4 コードの定型句 (§4.1 層 1)
      { "symbols": ["IV△7", "III7", "vi"], "weight": 3 }
    ],
    "idiomProbability": 0.45,
    "cadences": {                              // 終止形の好み (全終止/変格/偽終止/モーダル)
      "final": [{ "symbols": ["V7", "I△7"], "weight": 2 }],
      "half": [{ "symbols": ["V7"], "weight": 2 }]
    },
    "harmonicRhythm": {                        // "0.5"=2小節1コード, "2"=1小節2コード
      "default": { "1": 0.8, "2": 0.2 },
      "chorus": { "1": 0.5, "2": 0.5 }
    },
    "cliches": ["descending-bass"],           // 半音階クリシェ等の名前付き技法
    "borrowedChords": true                     // モーダル・インターチェンジ許可
  },
  "rhythm": {                                  // リズム語彙 (§4.1 層 2)。負値は休符
    "templates": { "4/4": [{ "beats": [1.5, 1.5, 1], "weight": 2 }] },
    "anacrusisProbability": 0.5,               // アウフタクト (先取音)
    "chorusDensityBias": 0.35                  // サビで音数の多いテンプレートを優先
  },
  "melody": {
    "leapProbability": { "default": 0.28, "chorusHead": 0.95 },
    "leapRangeSemitones": [7, 10],
    "afterLeapBias": "down",
    "contour": "leap-then-descend",
    "pedalPoint": false,                       // Pedal では true(逆ペダル)
    "motif": { "repeatProbability": 0.5 },     // モチーフ反復 (§4.1 層 3)
    "nonChordTones": {                         // 非和声音 (§4.1 層 5)
      "appoggiatura": 0.22, "suspension": 0.08, "chromaticPassing": 0.08
    },
    "registerShift": { "chorus": 4 }           // セクション別の音域シフト (半音)
  },
  "structure": {
    "phraseLengths": [4, 4, 4, 4],
    "irregularPhraseProbability": 0.0          // Modal では 0.3 等
  },
  "modulation": {                              // 転調傾向 (§4.1 層 6)
    "bridge": { "probability": 0.6, "intervals": [{ "semitones": 5, "weight": 2 }] }
  }
}
```

新フィールド (`idioms` / `cadences` / `harmonicRhythm` / `rhythm` / `motif` / `nonChordTones` / `registerShift` / `modulation`) はすべて省略可で、省略時は従来どおりの挙動 (マルコフ+V7 カデンツ+内蔵リズムテンプレート) にフォールバックする。

生成エンジンはこの定義を解釈する汎用インタープリタとして実装し、**エンジンとスタイル知識を分離**する。

#### スタイル追加の拡張レベル

| レベル | 対象者 | 方法 | 時期 |
|---|---|---|---|
| 1. 同梱スタイル | 開発者 | `dialects/` に JSON を追加(エンジン変更不要) | M3(4 スタイル) |
| 2. ユーザー定義 JSON | 上級ユーザー | 自作の定義ファイルを読み込み | M5 以前でも読み込み機能のみ先行実装可 |
| 3. GUI エディタ | 一般ユーザー | ダイアレクトエディタでパラメータを編集・保存・共有 | M5 |

**JSON で表現できる範囲の制約:** コード語彙・遷移確率・跳躍確率などのパラメータ調整と、既存の名前付き技法(`cliches` に指定する `descending-bass` 等)の組み合わせは JSON だけで自由に定義できる。一方、**技法アルゴリズム自体**(半音階クリシェや逆ペダルポイントの実装、あるいは対位法的な 2 声生成のような新技法)はエンジン側(`engine/`)のコード実装が必要。新技法を追加する際は、名前を付けて登録し JSON から参照できる形にすること(技法レジストリ方式)。

### 6.3 モジュール構成

```
src/
  engine/          # 純粋ロジック(UI 非依存・テスト対象)
    structure.ts   # セクション・フレーズ長生成
    harmony.ts     # コード進行生成(遷移表・借用・クリシェ)
    melody.ts      # メロディ輪郭・リズム割付
    accompaniment.ts
    explain.ts     # 適用ルールの注記データ生成
    rng.ts         # シード付き乱数
  dialects/        # ダイアレクト JSON 定義
  audio/           # 再生・WAV レンダリング
  export/          # SMF エンコーダ、テキスト出力
  ui/              # React コンポーネント
```

## 7. 非機能要件

- 1 曲(3 分相当)の生成は 1 秒以内(生成は同期処理で十分軽い想定)
- WAV レンダリングは 10 秒以内(Web Worker / OfflineAudioContext)
- 対応ブラウザ: Chrome / Edge / Safari / Firefox 最新版
- オフライン動作可(PWA 化は任意)

## 8. 権利・倫理上の注意

- 実在アーティストの録音・歌詞・メロディを一切収録/複製しない。実装するのは公知の音楽理論技法と統計的傾向のみ
- ダイアレクト名・id・UI 表記に実在の人物名を用いない。名称は技法に基づく抽象名(Chromatic / Modal / Pedal / Twilight 等)とする
- 生成曲の権利はユーザーに帰属する旨を明記

## 11. スタンドアロン化(将来: M6)

本アプリは完全クライアントサイド(§6.1)のため、Web 版のコードをほぼそのままデスクトップシェルに載せられる。

### 11.1 シェル候補

| 方式 | 特徴 |
|---|---|
| **Electron** | Chromium 同梱のため Web Audio / OfflineAudioContext の挙動が Web 版と完全一致。実績・情報量が多い。配布サイズ大(100MB 前後) |
| **Tauri** | OS 標準 WebView 利用で軽量(数 MB〜)。ただし Windows(WebView2)/ macOS(WKWebView)で描画・音声エンジンが異なり、Web Audio 周りのプラットフォーム別検証が必要 |
| **PWA** | インストール可能・オフライン動作。追加コストほぼゼロだがネイティブ連携は限定的。M6 を待たず早期に対応してもよい |

音の挙動一致を重視する場合は Electron を第一候補とする。配布サイズ優先なら Tauri(その場合 WebView 別の Web Audio 検証タスクを追加)。

### 11.2 スタンドアロン化で追加されるメリット・機能

- ネイティブのファイル保存ダイアログ(MIDI/WAV を DAW のプロジェクトフォルダ等へ直接保存)
- MIDI デバイス連携の強化(生成曲をハード音源や DAW へリアルタイム送信)
- ローカルのサウンドフォントファイル読み込み(§10 の音源容量制約がブラウザから解放される)
- オフライン完全動作・自動アップデート配布

### 11.3 設計上の前提(今から守ること)

- **§6.3 の `engine/` と `ui/` の分離を崩さない。** エンジンはブラウザ API 非依存の純粋 TypeScript に保つこと。これにより Electron メインプロセスやヘッドレス CLI(バッチ生成)でも同一コードを流用できる
- `audio/` と `export/` のみがプラットフォーム依存の境界となるため、ファイル保存などの I/O は薄い抽象層(Web 版: ダウンロード / デスクトップ版: ネイティブダイアログ)を挟んで実装する
